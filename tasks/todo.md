# 乩身值班系統 — 第四輪：經營養成 + 經濟 + 商店 + 信眾人生

## 需求（使用者）
1. 持續累積經營，全存 localStorage，真實累積，從零開始
2. 賺的錢可買：豪車/名錶/好酒/金飾/豪宅（便宜→超貴），諷刺私慾，給目標
3. 商店可查看購買（介面要像蝦皮商城，購物體驗很爽）
4. 信眾清單：看每位狀態，受歷史回答影響 → 回答改變他人生
5. 問神期間可看到信眾數值狀態
6. 精心設計信眾人設/故事線/多結局，多到玩不完
7. 難度：不是亂答就好，錢不夠經營不下去
8. 經濟：名聲低→信眾少、收入少；固定開銷；債務系統
9. 回訪事件：記仇上門鬧/色誘求婚/貢獻大錢…出乎意料，盡量有圖
10. 下班後「東區把妹」系統，滿足炫富慾
11. 需要圖的都列清單（提示詞＋存放位置）給使用者生圖

## 架構設計

### 持久化（localStorage）— `src/store.js`
- key `tangki.save.v1`：{ cash, debt, rep(名聲), day, shiftCount, owned[], believers{cid:{state}}, stats, datingScore, version }
- load/save/reset；每次重要動作自動存；版本遷移容錯。

### 經濟模型 — `src/economy.js`
- **名聲 rep**（0→∞）：決定每班信眾人數與出手大方度。低名聲→少人、少錢。
- **每班開銷 overhead**：基礎電費+法師薪+香火裝填，隨等級/名聲上升（規模越大開銷越大）。
- **收入**：香火（問答）+ 事件；扣 overhead = 淨利進 cash。
- **負債 debt**：cash 不足付 overhead → 可借錢（廟口錢莊），有利息；連續還不出 → 被討債（事件，扣更多/砸店）→ 撐不住 game over（廟封）。
- **難度曲線**：rep 低時收入 < 開銷，要靠提升滿意度/口碑拉 rep；亂答→滿意跌→rep 跌→更少人→死亡螺旋。
- 公式都集中此檔，好調平衡。

### 信眾人生系統 — `src/believers.js`（擴充）+ 存於 store
- 每位 cid 有 `life` 狀態：{ wealth, love, health, trust(對你), fate(結局傾向), visits, lastChoices[] }
- 你的每次選擇 push 進 lastChoices，改變 life 數值。
- 多結局：依 life 與你的建議 → ending（飛黃騰達/負債跳路/被劈腿/家破/幸福/出家/…）。
- **回訪事件**：trust 高→回來貢獻大錢/色誘求婚/送禮；trust 低→記仇上門鬧事/砸場/提告。回訪在後續班次隨機觸發。

### 問答深化 — `src/qa.js`（擴充）
- 問神期間信眾卡顯示即時數值（信任/財務/情緒小條）。
- 故事線擴充：每位 2–4 階段、多分支、依 life 狀態變化開場白。
- 多到玩不完：靠 多角色 × 多階段 × 分支 × life 影響後續 → 組合爆炸。

### 商店 — `src/shop.js` + 蝦皮風 UI
- 商品分類：豪車/名錶/好酒/金飾/豪宅，每類 cheap→insane 價格階梯。
- 蝦皮商城風介面：搜尋列、分類 tab、商品卡(圖/標題/價/「蝦幣回饋」假元素)、購物車、結帳動畫（爽感）。
- 純收藏炫耀，無數值加成。買完進「我的收藏」。
- 結算反諷：「這台車 = 可幫助 N 位信眾」。

### 把妹系統 — `src/dating.js`（下班後・東區）
- 用錢炫富把妹：開名車/戴名錶提升把妹成功率（這裡商品才有「作用」=炫富）。
- 小遊戲：選約會地點/送禮/對話，datingScore 累積，解鎖「對象」。
- 純諷刺爽感，不影響廟務經濟（花的是已賺的錢）。

### 主流程改造 — `src/main.js`
- 進入點：待機大廳（顯示 cash/rep/day/負債），可進：開始值班 / 商店 / 信眾清單 / 把妹 / 收藏。
- 一班 = 選神→準備→請神→執務(N 輪，N 由 rep 決定)→結算(算 overhead/淨利/rep 變動/還債)。
- 全程讀寫 store。

### UI 新頁面（HTML/CSS）
- 大廳 hub、商店(蝦皮風)、信眾清單(含狀態條與結局)、收藏櫃、把妹場景。
